# AGENTS.md

Guidance for AI agents and developers working on the **Claire Campbell PA**
real estate website.

## Project Overview

A single-page marketing site for a real estate agent (Claire Campbell PA,
Charles Rutenberg Realty, Orlando / Winter Park, FL). Its goals are to present
Claire's services and to capture leads via a contact form. There is no database
or authentication â€” it is a content + lead-capture site.

## Tech Stack

- **TanStack Start** (React 19) with file-based **TanStack Router**
- **Vite 7** build
- **Tailwind CSS 4** (configured via `@import "tailwindcss"` + `@theme` tokens,
  not a `tailwind.config.js`)
- **lucide-react** for icons
- **Netlify Forms** for the contact form
- Deployed on **Netlify** via `@netlify/vite-plugin-tanstack-start`

## Conventions

- **Design tokens** (colors, fonts) are defined once in `src/styles.css` under
  `@theme`. Use the resulting utility classes (`bg-forest`, `text-cream`,
  `text-gold`, `font-display`, etc.) rather than hard-coded hex values.
- **Business details** (phone, address, brokerage, service areas) come from
  `src/siteConfig.ts`. Do not hard-code them in components.
- The homepage is intentionally composed of small, self-contained section
  components (`Hero`, `About`, `Experience`, `Advantage`, `Reviews`,
  `Marketing`, `CtaBand`, `Contact`, `Footer`) inside `index.tsx`. Add or reorder
  sections there.
- Navigation is anchor-based (`#about`, `#contact`, â€¦) for smooth in-page
  scrolling â€” there is currently only one route.
- The `@/` import alias maps to `src/` (see `tsconfig.json`).

## Netlify Forms â€” Non-Obvious Requirements

Because TanStack Start renders forms client-side, Netlify cannot detect them
during the build. Two things keep the form working:

1. `public/__forms.html` contains a hidden static copy of the form with **every
   field name**. Netlify scans this at build time to register the `contact` form.
2. `ContactForm.tsx` submits via `fetch('/__forms.html', â€¦)` with
   `Content-Type: application/x-www-form-urlencoded` â€” **not** to `/`, which the
   SSR handler would intercept.

If you add or rename a form field, update **both** `ContactForm.tsx` and
`public/__forms.html`, then re-run the Netlify Forms enable script.

## Build & Run Notes

- Do **not** commit `src/routeTree.gen.ts` or run production builds manually â€”
  the route tree is generated by the router plugin and Netlify builds the site.
- Local dev: `netlify dev --port 8889`. Form submissions only process on a real
  Netlify deploy, not locally.
